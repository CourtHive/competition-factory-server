# Configuration

All configuration is via environment variables in the `.env` file. The server uses `dotenv` to load these at startup.

> **Important:** `dotenv` does **not** override environment variables already set in your shell. If a variable like `PG_USER` is exported in your shell profile, that value takes precedence over `.env`.

## Example `.env`

```env
APP_NAME='Competition Factory Server'
APP_MODE='development'
APP_PORT='8383'

JWT_SECRET='replace-with-a-truly-random-string'
JWT_VALIDITY='2h'

STORAGE_PROVIDER=leveldb

DB_HOST=localhost
DB_PORT=3838
DB_USER=admin
DB_PASS=adminpass

REDIS_URL=redis://localhost:6379
REDIS_TTL=28800000
```
