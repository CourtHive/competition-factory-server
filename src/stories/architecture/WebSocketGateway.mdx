# WebSocket Gateway

The server uses Socket.IO for real-time bidirectional communication with TMX clients. All tournament mutations, broadcasts, and notifications flow through the WebSocket gateway.

## Namespace

All TMX communication uses the `/tmx` Socket.IO namespace.

```typescript
// Client connection
const socket = io('http://localhost:8383/tmx');
```

## Authentication Flow

1. Client connects to `/tmx` namespace
2. Client sends `authenticate` message with JWT token
3. Server validates token, associates socket with user/provider
4. Subsequent messages are authorized based on user roles

## Mutation Acknowledgement

Each mutation request includes an `ackId`. The server responds with an `ack` message containing:

```json
{
  "ackId": "original-request-id",
  "success": true,
  "methods": [{ "method": "addEvent", "success": true }]
}
```

The client waits for this `ack` before applying mutations locally, ensuring server-client consistency.

## Broadcasting

When a client modifies a tournament, the server broadcasts the changes to all other clients connected to the same tournament. This enables real-time collaboration between multiple tournament directors.
